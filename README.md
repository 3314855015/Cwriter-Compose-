# CWriter - 创意写作助手

#### 写小说，和开发软件，从来不是一帆风顺，一蹴而就的事。

一款专为喜欢思考设计情节，注重读者沉浸感，对故事性与视角合理性有要求的作家创作的开源 Android 应用，帮助作者解决在创作长篇小说时“瞻前顾后”问题，以避免过多的纸质记述与备忘录使用。

## 我们主要解决了什么？

#### 一：便捷粘贴板 + 双重思维索引，设计了【沉浸式写作板】
- 把“遗忘前文”这个劣势，很大程度的变成了“帮作者代入主角”的优势，打破了“字数越多，效率反而越低”的死循环，有效避免了长篇写作中需要频繁“瞻前”（引申义，翻看前文）的问题

#### 二：多层级嵌套父子改故事线为类树形结构，开发了【故事树】
- 在保证故事扩展性的同时，简化了显示UI，同时，支持作家层层细分，避免了构思新故事替换旧故事时故事线有时需要作家整个重写情节的“顾后”（引申义，前边的情节设计要能够引出接下来的故事）问题

#### 三：妙用“段评”，改组了【伏笔管理】
- 在几乎完全没有影响文章显示效果的同时，解决了伏笔UI显示难，作家自己回忆难，伏笔创作后记忆难，过多伏笔管理难等问题，不采用恼人的snackbar提示，而是通过UI在创作过程中便捷暗示用户还有多少伏笔未回收，兼具“瞻前”“顾后”效果

## 技术栈

| 技术 | 说明 |
|------|------|
| **Kotlin** | 主要开发语言 |
| **Jetpack Compose** | 现代 Android UI 工具包 |
| **MVVM** | 架构模式，分离视图与业务逻辑 |
| **Kotlin Coroutines** | 协程，用于异步数据处理（待集成） |

## 最后，一些产品开发的心路历程，送给那些可能曾经同样被困扰，导致放弃小说创作的作家

- ### 嘛，首先呢，就分享分享，我是如何诞生出开发Cwriter的想法的呢？


   这个问题，其实可以追溯到我的高二时期，那时候，由于学习过于压抑，同时，受到当时小说热潮的影响，我萌发了一个“想要写一本独属于自己的故事”的愿望，可惜，那时候学业过重，而初入小说这个圈子，又恰巧遇上了“高人”指导（或许是同学？或许是视频？或许是评论？）总之，他告诉我说——写小说，必须得有大纲！而且，最好能把所有的关键情节，构思起来之后，再去动笔。
   
   结果可想而知，没有任何经验的我，构思的，完全就是流水线情节，除了使用一些当时流行的热梗以外，没有任何亮点，这直接导致，仅仅不到1w字左右，我就完全失去了对该本小说的热情……我看了都感觉无趣，何况他人？甚至，还给我留下了一个心理阴影——我是不是不是这块料？搞得后来，好长一段时间，我连翻开那本小说的勇气都没有，最后，还是身边一起看小说的朋友恰巧看到了我的小说，虽然，评价只有仅仅一句“不赖”，但，这就够了。

   后来，我把小说主角名字改成了对方的谐音，把作品名，改成了对方的小名，把这本记录着失败尝试的作品，送给了他，而后，开始反思——仅仅1w字，满腔热血想要构思独特故事的自己，怎么就会一蹶不振呢？

   首当其冲，故事大纲构造问题，一次性构思出所有的故事，完全不现实，其次，写作过程中，由于学业压力，效率有限的情况下，导致我哪怕只是一个人名，都需要频繁翻看前文，这让我莫名烦躁，最后，一些情节的呼应，字数起来以后，时间一旦拉长，连我这个“上帝”，居然也会忘记，当时写着的这个，到底是不是伏笔？是伏笔吧，它又是干什么的？所以，我就想，有没有这样一种工具，能帮我解决这些问题呢？

   这的想法，最终，在我大三系统学习Android开发后，化成了Cwriter的雏形
   

- ### 好的想法，得到的过程从不容易，贴近生活，明确需求，总会有着收获
